Analyzing Edits on HRC's page¶

Author: Ngina Kariuki, Ana Fernandez, Angelina Li Oct 29, 2017

Research questions: Did big news stories associated with Hillary Clinton cause more edit wars to occur on her page, as opposed to just adding new information while not necessarily causing edit wars? We will analyze this question by looking at the controversiality of HRC's page just before and right after controversial events occured.

Hypothesis: Our hypothesis is that big news stories about Hillary Clinton caused more edit wars to occur on her page.

Controversiality: We will use the number of reversions on a page as a proportion of the total number of edits on that page as a measure of how controversial a Wikipedia page is on any given day.

import requests
def list_of_diffsize(revisions):
    """Returns list of diffsize and list of datestring for the given revision"""
    
    urlbase = "https://en.wikipedia.org/w/api.php"
    diffsize_list, datestr_for_month = [], []
    
    for rev in revisions:
        req = requests.get(url=urlbase, params = { 'action': 'compare',
                                                    'format': 'json',
                                                    'prop': 'size|diffsize|user',
                                                    'fromrev': rev['parentid'],
                                                    'torev': rev['revid']})
        response = json.loads(req.content)
        try:
            diffsize_list.append(response['compare']['diffsize'])
        except KeyError:
            print(rev)
            diffsize_list.append(0)
            
        datestr_for_month.append(createDateTime(rev))
        
    return diffsize_list, datestr_for_month

6. Defining our controversiality metric¶

Now we will define a metric to assess how controversial a page is on any date.

In [31]:
def get_controversiality_index(revisions, get_time, get_format_time):
    """
    Return a dictionary mapping each day in our revisions data to how controversial that particular day is.
    """
    contr_index = {}
    
    for r in revisions:
        
        try:
            time = get_time(r)
            comment = r["comment"]
            
            # initialize to dictionary
            if time not in contr_index:
                str_time = get_format_time(r)
                contr_index[time] = {
                    "str_time": str_time, 
                    "edits": 0, 
                    "reversions": 0,
                    "contr_index": 0
                }

            contr_index[time]["edits"] += 1

            if "revert" in comment.lower():
                contr_index[time]["reversions"] += 1
        
        except KeyError:
            pass
        
    for time in contr_index:
        contr_index[time]["contr_index"] = float(contr_index[time]["reversions"]) / contr_index[time]["edits"]
        
    return contr_index
